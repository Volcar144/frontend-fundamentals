**UX (User Experience)** is how it *feels* to use a product — is it intuitive? Efficient?
**UI (User Interface)** is how it *looks* — typography, colour, spacing, components.

Great products need both.

---

## 1. Visual Hierarchy

Guide the user's eye using **size**, **weight**, and **contrast**. Every page should have one clear focal point.

| Level | Size | Weight | Usage |
|---|---|---|---|
| H1 | 48px | Bold | Page title (one per page) |
| H2 | 32px | SemiBold | Section headings |
| H3 | 24px | SemiBold | Sub-sections |
| Body | 16px | Regular | Main content |
| Small | 14px | Regular | Labels, captions |

---

## 2. Consistency

- Same button style for the same action
- Consistent spacing scale: **4, 8, 16, 24, 32, 48px**
- Consistent colour semantics: red = danger, green = success, yellow = warning
- Never change a familiar pattern without a clear reason

---

## 3. Feedback & Affordance

Users need to know what is clickable and what happened after they act.

```html
<button class="cursor-pointer rounded-lg bg-blue-600 px-4 py-2 text-white
               hover:bg-blue-700 hover:shadow-md
               active:scale-95
               focus-visible:ring-2 focus-visible:ring-blue-300
               transition-all">
  Submit
</button>
```

- **Loading states:** Show a spinner or skeleton while fetching
- **Success states:** Show a toast or green confirmation
- **Error states:** Show inline red error messages

## 4. Accessibility (a11y)

Accessibility is not optional — it's good design.

### Semantic HTML

```html
<!-- ❌ Bad -->
<div class="nav">...</div>
<div class="button" onclick="...">Click me</div>

<!-- ✅ Good -->
<nav>...</nav>
<button type="button">Click me</button>
```

### Colour Contrast

- Normal text: **4.5:1** minimum (WCAG AA)
- Large text (18px+): **3:1** minimum
- Check with: [webaim.org/resources/contrastchecker](https://webaim.org/resources/contrastchecker)

### ARIA Labels

```html
<button aria-label="Close modal"><XIcon /></button>
<img src="/hero.jpg" alt="Developer working at a laptop" />
<nav aria-label="Main navigation">
```

### Keyboard Navigation

- All interactive elements reachable by `Tab`
- Visible focus styles on every interactive element
- Modals trap focus inside when open

## 5. Colour Theory

Choose a three-part palette:

- **Primary** — brand colour, calls to action
- **Secondary** — accent, highlights
- **Neutral** — grays for text, backgrounds, borders

```html
<!-- Tailwind semantic pairings -->
<div class="bg-blue-50   text-blue-800   border border-blue-200">   Info    </div>
<div class="bg-green-50  text-green-800  border border-green-200">  Success </div>
<div class="bg-yellow-50 text-yellow-800 border border-yellow-200"> Warning </div>
<div class="bg-red-50    text-red-800    border border-red-200">    Error   </div>
```

---

## 6. White Space

White space is a design tool, not wasted space.

- Separate unrelated elements with **more** space
- Group related elements with **less** space (proximity)
- Let content breathe — don't cram

---

## 7. Mobile-First & Responsive

- Design smallest screen first, scale up
- Touch targets **≥ 44×44px**
- Don't rely on hover for key interactions
- Test on real devices

```html
<div class="flex flex-col gap-4 md:flex-row">
  <aside class="w-full md:w-64">Sidebar</aside>
  <main class="flex-1">Content</main>
</div>
```

## 8. Empty & Loading States

### Skeleton loader

```html
<div class="animate-pulse space-y-2">
  <div class="h-4 bg-gray-200 rounded w-3/4"></div>
  <div class="h-4 bg-gray-200 rounded w-1/2"></div>
</div>
```

### Empty state

```tsx
<div className="flex flex-col items-center justify-center py-16 text-center">
  <InboxIcon className="h-12 w-12 text-gray-400" />
  <h3 className="mt-4 text-lg font-semibold">No results found</h3>
  <p className="mt-2 text-sm text-gray-500">Try adjusting your search filters.</p>
  <button className="mt-4 btn-primary">Clear filters</button>
</div>
```

## Component Patterns

### Form best practices

```tsx
<div className="space-y-4">
  <div>
    <label htmlFor="email" className="block text-sm font-medium text-gray-700">
      Email address
    </label>
    <input
      id="email"
      type="email"
      className="mt-1 block w-full rounded-lg border border-gray-300 px-3 py-2
                 focus:border-blue-500 focus:outline-none focus:ring-1 focus:ring-blue-500"
    />
    <p className="mt-1 text-sm text-red-600">Please enter a valid email.</p>
  </div>
</div>
```

### Modal checklist

- Visible close button (`×`)
- Close on `Escape` key
- Focus trapped inside the modal
- Backdrop click closes it
- `aria-modal="true"` and `role="dialog"`
